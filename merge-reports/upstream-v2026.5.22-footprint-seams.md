## Scope

This addendum answers the narrower question the base merge report left open:
which new or improved upstream `v2026.5.22` seams can reduce Clawline's
OpenClaw core footprint now, or after small upstreamable work?

The matrix below is evidence-based. It does not treat existing Clawline
implementation as product truth, and it does not infer that core removal is safe
because a nearby upstream feature exists.

## Verdict Key

- `proven`: upstream seam already covers the Clawline carry with source/test
  evidence and can replace or shrink it without new product assumptions.
- `conditional`: upstream seam is relevant, but a named proof or small
  upstreamable adapter is required before removal.
- `blocked`: upstream seam is relevant, but current upstream contract prevents
  use for this footprint without broader product/design work.
- `not useful`: upstream seam is real, but it does not replace a current
  Clawline core carry.
- `unproven`: possible relationship exists, but evidence is insufficient.

## Summary

`v2026.5.22` does add or improve useful seams, but none prove broad Clawline
core-footprint removal by themselves.

Actionable now:

- `FS-002-plugin-surface-urls` is proven for Canvas-style hosted surfaces and
  should prevent future native-client compatibility carries, but it is not a
  Clawline runtime extraction seam.
- `FS-006-codex-app-server-control` is a useful local branch seam for fast-mode
  storage/control access, but it is not upstream-proven in `v2026.5.22`.
- `FS-013-bundled-plugin-package-boundary` remains the right branch shape: keep
  Clawline as an extension package and judge every remaining core carry
  independently.

Most promising conditional seams:

- `FS-003-task-ledger-rpc` / `FS-004-agent-harness-task-runtime` for task status
  and possibly generic completion delivery.
- `FS-007-tools-effective-catalog-invoke` for any Clawline tool inventory UI.
- `FS-011-config-schema-lookup` for config UI metadata once plugin config
  coverage is proven.

Negative findings, non-exhaustive:

- No upstream generic message-reference context contract was found.
- `FS-008-artifacts-rpc` is blocked for Clawline upload storage, `media://`
  handles, external `mediaUrl` fetching, and native attachment delivery. It may
  still be useful for generated transcript artifacts, but not as a current media
  footprint removal seam.
- Native Codex task runtime is private to the bundled Codex plugin.
- `defineToolPlugin` is not suitable for Clawline.
- Admin HTTP RPC is an operator-control seam, not a replacement for Clawline's
  device/message streaming runtime.
- The base merge report still retains Clawline SDK/runtime, media, heartbeat,
  subagent, packaging, and native-client carries. This addendum does not turn
  any of those categories into removable footprint without the row-specific proof
  above.

Merge readiness recommendation: unchanged. The merge branch is reviewable, but
core-footprint reduction should proceed seam-by-seam through the conditional
items above, not as part of the merge resolution itself.
