## Verification Rules

Each row below uses exactly one verification status:

- `satisfied/proven`: branch source/tests prove the specific seam claim. This does not imply broad Clawline core-footprint removal unless the row says so.
- `conditional`: branch source proves a relevant seam exists, but removal needs a named product proof, upstreamable adapter, or contract expansion.
- `unsatisfied`: branch source proves the seam does not currently replace the Clawline footprint described.
- `unproven`: evidence is insufficient to decide whether a useful removal seam exists.
- `out-of-scope`: the seam/finding is real, but it does not apply to Clawline's current OpenClaw core footprint.

I did not use current implementation behavior alone as product truth, and I did not treat tests, demos, examples, or harnesses as product evidence for removal. Tests are cited only as contract/sanity proof for source-backed claims.
